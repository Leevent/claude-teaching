Interactive Workshop

🧩 Skill 設計工坊

跟著五個步驟，設計出你的第一個 Skill。完成後可直接複製使用。

① 觸發條件 — 什麼時候啟動這個 Skill？

想想你工作中哪件事「手動做第三次，就值得做成 Skill」

提示：描述一個具體的工作場景，越具體越好

② 輸入規格 — Skill 需要你提供什麼？

列出 Skill 啟動時需要的資料，越明確越不容易出錯

③ 執行流程 — Step by step 怎麼做？

把你腦中的工作流程寫下來，Claude 會照著這個順序執行

④ 輸出格式 — 最終交付什麼？

明確定義產出的格式和結構，避免每次結果長得不一樣

官方建議：給 3-5 個範例效果最好。這裡先放 1 個核心範例。

⑤ 品質護欄 — 怎麼確保產出品質？

Red Line Rules（紅線規則）：告訴 Claude「不做什麼」比「做什麼」更重要

Skill Maturity Model

📈 5 級 Skill 成熟度

你目前在哪個等級？這堂課帶你從 L1 走到 L3-L4

L1
結構化提示 Structured Prompt
用五元素框架（R-T-C-F-Q）寫出清楚的指令。這是 A 班教的基礎功，大多數人停在這裡。每次都要重新寫 prompt，無法複用。
L2
動態提示 Dynamic Prompt
加入變數和條件判斷，同一個 Skill 能適應不同情境。例如：根據「產品類型」自動切換不同的文案風格和平台格式。
L3
工具調用 Tool Calling
Skill 能透過 MCP 呼叫外部工具：讀你的 Calendar、查 Notion、操作 Sheets。從「純文字產出」進化到「連接真實世界」。
L4
多步驟 Multi-step
Skill 自動串起多個步驟：先蒐集資料 → 分析趨勢 → 撰寫報告 → 推送到指定位置。每一步的輸出成為下一步的輸入。
L5
編排調度 Orchestration
多個 Skill 協作、SubAgent 並行處理。一個指令拆分給多個 Agent：一個做競品分析、一個寫文案、一個做 SEO 優化。這是隱藏班的領域。

FAQ

常見問題

我沒上過 A 班，可以直接報 B 班嗎？ ▾
如果你已經熟悉 Claude 的 Projects、Artifacts 功能，可以直接報名。不確定的話，可以先做「Claude 使用力等級測驗」，看看自己在哪個階段。測驗結果在「中階」以上就能跟上。
上課需要付費版 Claude 嗎？ ▾
建議使用 Claude Pro 以上方案。課程中的 MCP 串接和部分進階功能需要 Pro 版。課堂上會先完整示範，你可以之後再決定是否升級。
MCP 串接需要寫程式嗎？ ▾
不需要！MCP 的設定是在 Claude 桌面版介面中完成的，課堂上會一步步帶你操作。你只需要有 Google 帳號和 Notion 帳號就好。
Skill 和 Projects 有什麼不同？ ▾
Projects 是你的「AI 工作空間」，讓 Claude 記住你的背景和偏好——像一個跟你共事的同事。Skill 是你設計的「AI 工作 SOP」，一個指令就能觸發完整的工作流程——像一份自動化流程表。兩者搭配使用，效果最好。
錯過直播怎麼辦？ ▾
會提供課後錄影回放，保留 14 天。這個網站上的互動工具（Skill 設計工坊、MCP 指南、系統畫布）不受時間限制，隨時可以使用。
MCP 和 n8n / Zapier 有什麼不同？ ▾
MCP 是 Claude 的「手和眼」，即時互動——你下指令，Claude 當場幫你操作。n8n / Zapier 是排程型自動化，設定好後在背景自動觸發。不是替代關係，是互補：MCP 做臨時決策，n8n 做重複排程。
我已經很熟 Claude 了，B 班對我還有幫助嗎？ ▾
如果你已經在用 Claude Code 或 API，B 班能幫你整理 Skill 設計框架、建立 MCP 架構思維，並作為隱藏班（Claude Code 進階班）的前置知識。課程包含 Anthropic 官方設計模式（Red Line Rules、Data Hierarchy），即使是重度使用者也能從系統化框架中獲益。
「隱藏班」是什麼？ ▾
隱藏班是這個系列的進階班，專注 Claude Code 開發——讓 AI 直接幫你寫程式、建系統、跑自動化。B 班結業後會開放報名資訊。三堂課的升級路徑：A 班（學會溝通）→ B 班（設計工作流）→ 隱藏班（建造系統）。
MCP 串接公司資料安全嗎？ ▾
MCP 的權限由你控制——Google Calendar 和 Notion 都是 OAuth 授權，你可以選擇分享哪些日曆或頁面。Claude 不會永久儲存你的資料。課堂上會說明哪些資料適合串接、哪些公司機密資料建議保留不連，以及權限管理的最佳做法。
